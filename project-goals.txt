**Project Overview & Design Document**

## 1. Introduction
### 1.1 Purpose
The purpose of this project is to develop a flexible, multi-tenant estimating software designed for professionals who estimate jobs without detailed prints. Unlike traditional "take-off" software, this tool is intended for small contractors and individuals needing a dynamic, customizable estimation system.

### 1.2 Target Audience
- Small contractors and individual estimators.
- Businesses needing a lightweight estimating tool.
- Users who prefer an open-source, self-hosted solution.

### 1.3 Key Problems Solved
- **Dynamic Estimation:** Instead of requiring predefined components for every variation (e.g., different wire lengths), this system allows unit-based cost and labor calculations.
- **Editable Packages:** Users can modify packages at the estimate level without affecting base components.
- **Open-Source:** Enables community contributions and broad accessibility.

---

## 2. Core Features
### 2.1 Customization
- Users can define their own labor and material costs.
- Packages and components can be modified per estimate.

### 2.2 User Roles (Dynamic)
- **Admin:** Full system access.
- **Estimator:** Can create and modify estimates.
- **Manager:** Oversees estimations but with limited admin access.

### 2.3 Multi-Tenancy
- Self-hosted instances should support multiple departments or businesses without cross-over.

### 2.3.1 Multi-Tenancy Implementation
- Row-level multi-tenancy will be implemented, where all tenants share the same database tables.
- Each record will be associated with a tenant_id to ensure proper data isolation.
- Middleware will enforce tenant isolation throughout the application.
- Query scoping will be implemented to prevent data leakage between tenants.

### 2.4 Data Structure
- **Items:** Can exist in three forms:
  1. Direct materials (attached directly to estimates)
  2. As part of assemblies
  3. As base templates for future use
- **Assemblies:** Groups of items with predefined quantities
- **Packages:** Collections of assemblies that form a complete estimate
- **Labor Rates:** Customizable labor costs per unit
- **Status Tracking:** Estimates can move through statuses (Draft, Submitted, Approved, etc.)
- **Version Control:** Not included in V1 but planned for future implementation

### 2.4.1 Package and Assembly Structure
- When packages or assemblies are added to an estimate, complete copies will be created rather than delta-based modifications
- This approach ensures calculation accuracy and simplifies the data model
- Modified items within estimates will be completely independent from their source items
- Changes to base packages or assemblies will not affect existing estimates

### 2.5 Workflow
1. Create new estimate
2. Add direct materials as needed (for one-off items)
3. Select and add predefined packages
4. Add additional assemblies as needed
5. Modify any items within the estimate without affecting the base templates
6. Generate exportable estimates (PDF, CSV, etc.)

### 2.6 Export & Collaboration
- **Exports:** Estimates can be exported in common formats.
- **Collaboration:** Not in V1, but design should accommodate future collaboration features.

### 2.6.1 Export Formats
- CSV export will be supported for data manipulation in spreadsheet software.
- PDF export will be implemented for professional, client-ready documentation.
- Export functionality will include options for customizing the output format and included data.

### 2.7 User Onboarding
- A streamlined onboarding process will guide new users through initial setup.
- Process will include admin registration, company/tenant setup, and initial data configuration.
- Optional demo data can be provided to help users understand the system's capabilities.
- Basic tooltips and guidance will be provided for key features.

---

## 3. Business Model
- **Free self-hosting** for individual users and businesses.
- Potential future **paid hosted version**, but monetization is not a priority for initial development.

---

## 4. Tech Stack
- **Backend:** Laravel (PHP Framework)
- **Frontend:** Livewire (integrated with Laravel)
- **Database:** MySQL
- **Caching & Queues:** Redis
- **Hosting:** Self-hosted and potential cloud-based version
- **Offline Support:** Possible if self-hosted on a local network

---

## 5. Roadmap & Future Enhancements
### V1 Development Priorities
- Basic estimation with dynamic components and packages.
- Role-based user management.
- Export functionality (CSV and PDF).
- Multi-tenant support with row-level isolation.
- Simple user onboarding process.

### Future Features
- Version control for estimates.
- User collaboration tools.
- Supplier pricing integration.
- AI-powered estimating suggestions.
- Offline support capabilities.

## 6. Community Contributions
- Open-source from the start.
- Users can contribute features and improvements.

---

## Conclusion
This project is designed to be an accessible, highly customizable estimating tool that addresses key pain points in the industry. By focusing on flexibility and simplicity, it offers a streamlined solution for independent estimators and small businesses.
